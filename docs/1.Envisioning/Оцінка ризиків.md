## 1. МОЖЛИВІ РИЗИКИ

### 1.1. Вступ
Метою фази виявлення ризиків є створення проектною групою списку наявних *ризиків проекту*. Цей список має в максимально-можливому ступені охоплювати всі чинники, які впливають на проект.

Вихідними даними фази виявлення ризиків є *наявні знання* як про загальні, так і про специфічні для конкретного проекту ризики, пов'язані з бізнесом, технологіями, організаціями і зовнішніми умовами.

### 1.2. Список ризиків

| № | Найменування ризику | Коментар |
|:--:|---|---|
| 1 | Сервер недоступний | Користувач не може отримати нотатки |
| 2 | Помилки API | Нотатки не зберігаються |
| 3 | Помилка підключення до MongoDB | Дані не записуються у базу |
| 4 | Втрата даних у БД | Нотатки зникають |
| 5 | Помилки MongoDB запитів | Пошкодження або некоректна вибірка даних |
| 6 | Повільні запити до MongoDB | Застосунок працює повільно |
| 7 | Неправильна робота Markdown рендеру | Текст відображається некоректно |
| 8 | Проблеми адаптивності UI | Поганий UX на мобільних пристроях |
| 9 | Помилки експорту .md | Користувач не може завантажити файл |
| 10 | Проблеми авторизації або доступу | Користувач не може працювати з нотатками |

## 2. КЛАСИФІКАЦІЯ РИЗИКІВ

| Першопричина | Умова | Наслідок | Нанесений збиток |
