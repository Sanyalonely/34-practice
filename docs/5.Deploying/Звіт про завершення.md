## 1. РЕАЛІЗАЦІЯ БАЧЕННЯ

На початковому етапі проєкт *NotatkaHUB* передбачав створення клієнтського веб-застосунку для роботи з нотатками у форматі Markdown із базовими можливостями (створення, перегляд, редагування, видалення), локальним збереженням та простим інтерфейсом.

### Досягнутий результат:
У фінальній версії було реалізовано:
- Інтерфейс на React + TypeScript
- CRUD-операції з нотатками
- Підтримка Markdown
- Перемикання між режимами редагування і перегляду
- Світла та темна тема
- Адаптивний дизайн
- Збереження даних:
  - MongoDB
- Експорт нотаток у форматі `.md`
- Розгортання:
  - Frontend — Vercel
  - Backend — Render

### Основні відмінності від початкового бачення:
- Додано серверну частину (MongoDB), що не планувалося спочатку
- Додано реєстарцію та авторизацію
- Інтерфейс залишився частково спрощеним
- Експорт реалізовано без архівації
- Терміни розробки були перевищені через складність інтеграції бекенду

**Висновок:** фінальне рішення перевищило початкове бачення за функціональністю, але має спрощений інтерфейс.

## 2. ЗМІНИ, ЩО ВПЛИНУЛИ НА БАЧЕННЯ

### 2.1. Зміни в баченні
- Додано бекенд (MongoDB + API)
- Відмовились від Firebase
- Додано темну/світлу тему
- Перейшли до клієнт-серверної архітектури

**Причини:**
- Бажання зробити проєкт більш масштабованим
- Практика повного стеку технологій

---

### 2.2. Зміни в команді
- Перерозподіл ролей:
  - Frontend (React)
  - Backend (Node.js + MongoDB)
  - UI/UX і тестування
- Один із учасників зменшив участь у проєкті

**Причини:**
- Різний рівень підготовки
- Обмеження часу

**Рекомендації:**
- Чітко визначати ролі
- Використовувати таск-менеджери
- Проводити регулярні зустрічі

---

### 2.3. Зміни в процесах на стороні замовника
- Замовник дозволив використання серверної частини
- Вимоги стали більш гнучкими

**Вплив:**
- Розширення функціональності
- Збільшення складності

**Рекомендації:**
- Фіксувати вимоги перед початком
- Вести контроль змін

---

### 2.4. Зміни в процесі роботи над проєктом
- Перехід до клієнт-серверної архітектури
- Додано API-запити
- Впроваджено регулярні онлайн-зустрічі

**Вплив:**
- Покращення якості рішення
- Збільшення часу розробки

---

### 2.5. Зміни в планах
- Більше часу витрачено на:
  - бекенд
  - інтеграцію MongoDB
- Менше — на дизайн

**Причини:**
- Недооцінка складності
- Нові технології

**Вплив:**
- Затримка термінів
- Покращення функціональності

---

### 2.6. Зміни в специфікаціях
- Додано API
- Змінено логіку збереження
- Спрощено UI

**Вплив:**
- Краща масштабованість
- Вища складність

---

### 2.7. Зміни у графіках
- Backend зайняв більше часу
- Frontend завершено швидше
- Загальний дедлайн зміщено

**Вплив:**
- Перерозподіл задач
- Затримка релізу
